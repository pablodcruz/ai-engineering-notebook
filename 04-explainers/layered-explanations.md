# Layered Explanations

Use this file to practice explaining technical ideas at different levels without turning the repo into a prep packet.

## LLMs

Engineering view: A model API generates outputs from instructions, input context, parameters, and surrounding application logic.

Product view: A system that can draft, summarize, classify, and transform language-based work when given clear instructions and useful context.

Plain-language view: A powerful text engine that is good at patterns, but still needs context and checks.

## RAG

Engineering view: Retrieval plus generation: search relevant chunks, inject them into the prompt, generate grounded output, and evaluate retrieval and answer quality.

Product view: A way to make AI answer from trusted material instead of relying only on general model knowledge.

Plain-language view: Let the app look things up before it answers.

## Agents

Engineering view: A model-driven loop that can choose tools, observe outputs, and continue toward a constrained goal.

Product view: An assistant that can take guided steps across systems, not just answer a single question.

Plain-language view: A model plus tools, rules, and checkpoints.

## Prompt Engineering

Engineering view: Task specification, context management, examples, constraints, output schema, and evaluation.

Product view: Giving the AI a clear job, the right background, and a useful answer format.

Plain-language view: Better instructions, better context, better checks.
